# コース3: skills実践編

skillsの使い方を学ぶコースです。

## skillsとは

skillsは特定のタスクを実行するための再利用可能な手順です。subagentがskillsを使って作業を行います。

| skill | 説明 |
|-------|------|
| `/fix-bug` | バグ修正ワークフロー |
| `/refactor` | リファクタリングワークフロー |
| `/commit` | コミットメッセージ生成とコミット |
| `/review-code` | コードレビュー |

## 演習1: 新しいバグを見つける

このアプリにはもう1つバグがあります。

### 現象

1. タイトルが空のTODOを作成できてしまう

```bash
curl -X POST http://localhost:8080/api/todos \
  -H "Content-Type: application/json" \
  -d '{"title": ""}'
```

## 演習2: /fix-bug でバグを修正

### skillを実行

```
/fix-bug
```

Claude Codeが以下の情報を確認します：
- バグの内容
- 再現手順

### 指示

```
タイトルが空の場合はTODOを作成できないようにバリデーションを追加してください
```

## 演習3: /refactor でコードを改善

バグ修正後、コードの品質を改善しましょう。

### skillを実行

```
/refactor
```

### 指示

```
TodoControllerのエラーハンドリングを改善してください。
バリデーションエラー時に適切なエラーメッセージを返すようにしてください。
```

## 演習4: /commit で変更をコミット

変更をコミットしましょう。

### skillを実行

```
/commit
```

Claude Codeが以下を行います：
1. 変更内容の分析
2. 適切なコミットメッセージの生成
3. コミットの実行

## 演習5: /review-code でセルフレビュー

最後に、自分の変更をレビューしましょう。

### skillを実行

```
/review-code
```

レビュー観点：
- コード品質
- セキュリティ
- パフォーマンス
- ベストプラクティス

## まとめ

このコースで学んだこと：
- `/fix-bug` でバグを修正する方法
- `/refactor` でコードを改善する方法
- `/commit` で変更をコミットする方法
- `/review-code` でセルフレビューする方法

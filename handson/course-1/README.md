# コース1: 基本操作編

Claude Codeとの対話に慣れるためのコースです。

## 前提条件

- Claude Codeがインストール済み
- ターミナルで `claude` コマンドが実行可能

## 演習1: Claude Codeの起動

1. ターミナルを開く
2. `handson/spring-boot-app` ディレクトリに移動
3. 以下のコマンドでClaude Codeを起動

```bash
cd handson/spring-boot-app
claude
```

## 演習2: 会話でコードを理解する

Claude Codeに以下の質問をしてみましょう。

### 質問1: アプリの概要を聞く

```
このSpring Bootアプリは何をするアプリですか？
```

Claude Codeがコードベースを分析して回答してくれます。

### 質問2: ファイルを指定して質問

```
@TodoController.java の役割を教えてください
```

`@ファイル名` でファイルを指定できます。

### 質問3: 特定のメソッドについて質問

```
TodoServiceのgetAllTodosメソッドは何をしていますか？
```

## 演習3: /コマンドを試す

Claude Codeには便利なコマンドがあります。

### ヘルプを表示

```
/help
```

利用可能なコマンド一覧が表示されます。

### よく使うコマンド

| コマンド | 説明 |
|----------|------|
| `/help` | ヘルプを表示 |
| `/clear` | 会話履歴をクリア |
| `/status` | 現在の状態を表示 |

## 演習4: ファイルの指定方法

複数のファイルを指定する方法を試してみましょう。

```
@TodoController.java と @TodoService.java の関係を説明してください
```


## 演習5: contextの管理

Claude Codeとの会話が長くなると、「context（文脈）」が蓄積されます。

### contextを確認する

```
/context
```

現在のcontext量が表示されます。

### contextが増えるとどうなる？

- 会話履歴、読み込んだファイル、実行結果などがcontextに含まれる
- **contextが増えすぎると精度が低下する**
- 関係ない情報が混ざり、回答がブレやすくなる

### /reset でリセットする

タスクが一区切りついたら、contextをリセットしましょう。

```
/reset
```

**使いどころ：**
- 別の作業に移るとき
- 回答がおかしくなってきたとき
- 長時間の作業後

> **Tips:** こまめに `/reset` することで、常に高い精度を維持できます。

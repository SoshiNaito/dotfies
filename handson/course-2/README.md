# コース2: subagent活用編

subagentを使った調査・修正を体験するコースです。

## subagentとは

subagentは特定の役割を持った専門エージェントです。Claude Code本体がオーケストレーターとして、適切なsubagentに作業を委譲します。

| subagent | 役割 |
|----------|------|
| `explorer` | コードベース調査（変更なし） |
| `debugger` | エラー調査・修正 |
| `tester` | テスト設計・実装・実行 |
| `implementer` | コード実装（新規・修正） |
| `reviewer` | コードレビュー、改善提案 |

## 演習1: explorerでコードベースを調査

### 指示を出す

```
explorerを使って、TODO取得エンドポイントがどのように実装されているか調査してください
```

explorerは以下の情報を調査します：
- エンドポイントの場所
- 呼び出し関係
- データの流れ

## 演習2: バグを発見する

このアプリには意図的にバグが仕込まれています。

### 現象

1. TODOを作成
2. TODOを完了にする
3. 一覧取得すると、完了したTODOが表示されない

### explorerで調査

```
explorerを使って、getAllTodosメソッドを調査してください。なぜ完了済みTODOが一覧に表示されないのでしょうか？
```

## 演習3: debuggerでバグを修正

バグの原因がわかったら、修正しましょう。

### 指示を出す

```
debuggerを使って、getAllTodosメソッドのバグを修正してください。
全てのTODO（完了/未完了問わず）が取得できるようにしてください。
```

debuggerは以下の手順で修正します：
1. 問題の特定
2. 修正案の提示
3. コードの修正
4. 動作確認

## 演習4: testerでテストを追加

修正後、テストを追加してバグが再発しないようにしましょう。

### 指示を出す

```
testerを使って、getAllTodosが完了済みTODOも含めて返すことを確認するテストを追加してください
```

testerは以下を行います：
1. テストケースの設計
2. テストコードの実装
3. テストの実行

## 演習5: 修正を確認

全てのテストが通ることを確認しましょう。

```bash
./gradlew test
```

## まとめ

このコースで学んだこと：
- explorerでコードベースを調査する方法
- debuggerでバグを修正する方法
- testerでテストを追加する方法

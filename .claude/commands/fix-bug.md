バグ修正ワークフローを開始します。

バグ情報: $ARGUMENTS

手順:
1. **バグの理解**
   - エラーメッセージを分析
   - 再現手順を確認
   - 関連コードを特定

2. **原因調査**
   - スタックトレースを追跡
   - 関連ファイルを検索
   - 最近の変更を確認 (`git log`)

3. **テスト作成（TDD）**
   - バグを再現するテストを作成
   - テストが失敗することを確認

4. **修正実装**
   - 最小限の変更で修正
   - 副作用がないか確認

5. **検証**
   - テストがパスすることを確認
   - 関連テストも実行
   - リグレッションがないか確認

6. **コミット**
   - `fix(<scope>): <description>` 形式でコミット
   - 関連Issue番号を含める

## 調査コマンド
```bash
# エラーログ確認
npm run dev 2>&1 | grep -i error

# 最近の変更
git log --oneline -20

# 特定のエラーを検索
grep -r "ErrorMessage" src/
```

今すぐバグの調査を開始してください。

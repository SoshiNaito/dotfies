`~/.claude/CLAUDE.md` の内容をdotfilesリポジトリに同期し、PRを作成します。

## 手順

1. **ソースファイルの読み取り**
   - `~/.claude/CLAUDE.md` を読み取る
   - ファイルが存在しない場合はエラーを報告して終了

2. **ターゲットファイルの読み取り**
   - dotfilesリポジトリの `.claude/CLAUDE.md` を読み取る
   - 現在の内容を確認

3. **差分の確認**
   - ソースの内容がすでにターゲットに含まれているか確認
   - 新しい内容がある場合のみ続行
   - 変更がない場合は「同期済みです」と報告して終了

4. **変更内容のプレビュー**
   - 追記される内容をユーザーに表示
   - 確認を求める

5. **ブランチの作成**
   - `sync/claudemd-YYYYMMDD-HHMMSS` 形式でブランチを作成

6. **ファイルの更新**
   - ターゲットファイルの末尾に以下の形式で追記:
   ```markdown

   [ソースファイルの内容]

   ```

7. **コミットの作成**
   - Conventional Commits形式でコミット
   - 例: `docs(claude): ~/.claude/CLAUDE.mdの設定を同期`

8. **PRの作成**
   - PRタイトル: `docs(claude): ~/.claude/CLAUDE.mdの設定を同期`
   - PRボディに以下を含める:
     - 変更の目的（なぜこの同期が必要か）
     - 追加された設定の概要
     - 影響範囲
     - 誰による変更か

## PRボディのテンプレート

```markdown
## 概要
CLAUDE.mdのルールを追加します。

## 変更理由
[ユーザーに変更理由を質問し、ここに記載]

## 追加される設定
[追記内容の要約]

## 影響範囲
- このdotfilesを使用する全ての環境に反映されます
- 既存の設定は保持され、末尾に追記されます
```

## 注意事項
- 同期前に必ず差分を確認してください
- 機密情報（APIキー、パスワード等）が含まれていないか確認してください
- PRを作成する前に変更理由をユーザーに質問してください

今すぐ同期を開始してください。

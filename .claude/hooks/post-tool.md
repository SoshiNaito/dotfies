# ツール実行後フック

## 用途
特定のツール実行後に自動的に追加処理を行います。

## トリガー条件

### ファイル変更後
ファイルを編集・作成した後:
- 関連テストの存在確認
- Lintチェックの提案
- 型チェックの提案

### テスト実行後
テストが失敗した場合:
- 失敗したテストの詳細を表示
- 関連コードの場所を特定
- 修正提案

### Git操作後
コミット後:
- コミット内容のサマリー表示
- プッシュの提案

プッシュ後:
- PR作成の提案
- CI/CDステータスの確認方法を案内

### ビルド後
ビルド失敗時:
- エラーの詳細表示
- 解決策の提案

ビルド成功時:
- 成果物の場所を表示
- 次のステップを提案

## 通知形式

### 成功時
```
✅ [操作名] 完了
   結果のサマリー
```

### 警告時
```
⚠️  [操作名] 完了（警告あり）
   警告の詳細
```

### 失敗時
```
❌ [操作名] 失敗
   エラーの詳細
   💡 提案: 解決策
```

## 自動提案

### ファイル作成後
```
📁 新しいファイルを作成しました: src/utils/helper.ts

💡 提案:
- テストファイルを作成しますか？ → src/utils/helper.test.ts
- エクスポートをindex.tsに追加しますか？
```

### 大きな変更後
```
📊 変更サマリー:
- 変更ファイル: 5
- 追加行数: +150
- 削除行数: -30

💡 提案:
- テストを実行してリグレッションを確認
- コードレビューを依頼
```
